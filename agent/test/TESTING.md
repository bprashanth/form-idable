# AI Verification — Manual Testing

This documents the two-step species correction pipeline that was validated manually.
The fuzzy-only path is what runs in the API today. The AI step is documented here
for a future agent to integrate after verifying it works end-to-end via CLI.

## Inputs

| File | Description |
|---|---|
| `agent/test/output.xlsx` | Excel output from Textract pipeline |
| `agent/test/segmented_000.jpg` | Original form photograph used to produce the xlsx |
| `agent/test/fuzz_proposals.json` | Output of Step 1 (generated by fuzz_match.py) |
| `agent/test/gemini_output.txt` | Saved output of Step 2 for reference |

## Step 1 — Fuzzy match (no AI, instant)

```bash
# Run from repo root
python3 agent/test/fuzz_match.py
# Reads:  agent/test/output.xlsx + agent/data/species_name.csv
# Writes: agent/test/fuzz_proposals.json
# Prints: table of (original, candidate, score%)
```

Output format:
```json
{
  "proposals": [
    {"original": "Kage",  "candidate": "Litsea wightiana",        "matched_key": "kage",       "score": 100.0},
    {"original": "Mine",  "candidate": "Neolitsea scrobiculata",   "matched_key": "mine",       "score": 100.0},
    {"original": "100ml", "candidate": "Magnolia nilagirica",      "matched_key": "magnil",     "score": 36.4}
  ]
}
```

## Step 2 — Gemini verification (AI, requires gemini CLI auth)

Gemini receives the fuzz proposals and the form image and acts as a verifier:
confirm good matches, null obvious noise, correct wrong matches by reading the handwriting.

### Via run_agent.sh

```bash
# Build prompt (template + fuzz JSON appended)
cat agent/prompts/species_verify.md \
    agent/test/fuzz_proposals.json \
    > /tmp/verify_prompt.md

# Run
agent/scripts/run_agent.sh \
  /tmp/verify_prompt.md \
  agent/test/fuzz_proposals.json \
  agent/test/segmented_000.jpg
```

### Directly via gemini CLI

```bash
cat /tmp/verify_prompt.md | gemini \
  @agent/test/fuzz_proposals.json \
  @agent/test/segmented_000.jpg
```

Expected output format (see `gemini_output.txt` for full example):
```json
{
  "corrections": [
    {"original": "Kage",   "corrected": "Litsea wightiana"},
    {"original": "100ml",  "corrected": null},
    {"original": "/cape",  "corrected": null}
  ]
}
```

## What changed between Step 1 and Step 2

From the saved run in `gemini_output.txt` vs `fuzz_proposals.json`:

| Entry | Fuzzy candidate | Score | Gemini | Note |
|---|---|---|---|---|
| `the` | NA | 75% | null | noise |
| `Kaji` | NA | 67% | null | noise |
| `them` | NA | 67% | null | noise |
| `cape` | Daphniphyllum | 60% | null | wrong match rejected |
| `/cape` | Daphniphyllum | 55% | null | noise |
| `kinlcar` | NA | 55% | null | noise |
| `100ml` | Magnolia | 36% | null | identified as DBH measurement |

Everything at ≥61% that fuzzy proposed was confirmed unchanged.
Gemini added zero new corrections that fuzzy missed — its value is noise filtering only.

## Integrating Step 2 into the API

When integrating, the agent should:

1. **Verify the CLI works first.** Run Steps 1+2 above and confirm the output matches
   `gemini_output.txt`. The main failure mode is gemini CLI auth — `run_agent.sh` uses
   `XDG_CONFIG_HOME=$(mktemp -d)` which clears cached credentials, causing a 60s timeout.
   Resolve auth before proceeding (e.g. `GEMINI_API_KEY` env var or persistent auth).

2. **Update `services/llm.py`** to call `run_agent.sh` with:
   - Prompt: `species_verify.md` + fuzz proposals JSON appended
   - Files: `fuzz_proposals.json` (temp), original form image (passed from request)

3. **Update `/agent/check-species` in `routers/checks.py`** to:
   - Accept an optional `image` file upload alongside `xlsx`
   - Run fuzzy match first (always)
   - If image present and gemini available: pass fuzz output + image to `llm.py`
   - Return merged result (gemini output takes precedence where available)

4. **Store the original image** in the PWA: `croppedImage` already exists in
   `useFormStore.js` but is not currently passed to the agent. Wire it through
   `ResultView.vue` → `agentPost('/agent/check-species', fd)` as an additional
   `fd.append('image', croppedImageBlob, 'form.jpg')`.
